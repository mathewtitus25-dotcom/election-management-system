\chapter{TESTING}

Testing is an essential phase of software development that validates the correctness, security, and reliability of the VoteDesk system. The testing strategy ensures that all modules -- including voter registration, BLO approval, candidate management, vote casting, and result publication -- function as specified. Both Unit Testing and Integration Testing methodologies were applied.

\section{Testing Results}

All 18 unit test cases and 8 integration test cases were executed successfully. The system correctly enforces all business rules including voter age restrictions, duplicate vote prevention, panchayat boundary enforcement, and election state gating. No critical failures were detected during the testing phase. Minor issues identified during early testing -- including session confusion during multi-role login and OTP expiry timing -- were resolved before the final submission.
